# AttackProbe 六个月工作计划

> 版本: v1.0 | 创建日期: 2025-01-15 | 状态: 待评审

## 总体目标

在六个月内完成：
1. 4-5人研发团队组建
2. 确定产品规划路径、技术栈选型、版本迭代计划
3. 输出 **AttackProbe 1.0版本**
4. 完成 **不少于3个客户** 的部署和试用

## 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 招聘困难 | 团队组建延迟 | 提前启动招聘，扩大渠道，备选外包 |
| 需求变更 | 开发返工 | 需求评审严格把关，变更控制流程 |
| 技术难点 | 进度延迟 | 复用llm-security-bas已有代码，降低技术风险 |
| 客户配合度低 | 试用延迟 | 提前沟通，多准备备选客户 |
