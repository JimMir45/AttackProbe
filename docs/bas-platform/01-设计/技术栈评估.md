## 3. 数据库选型

### 候选方案对比

| 维度 | SQLite | PostgreSQL | MySQL |
|------|--------|-----------|-------|
| **部署复杂度** | ⭐⭐⭐⭐⭐ 零配置 | ⭐⭐⭐ 需安装 | ⭐⭐⭐ 需安装 |
| **性能(小数据量)** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **性能(大数据量)** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **运维成本** | ⭐⭐⭐⭐⭐ 无 | ⭐⭐⭐ 需DBA | ⭐⭐⭐ 需DBA |
| **已有代码复用** | ✅ llm-security-bas用SQLite | ❌ 需改造 | ❌ 需改造 |

### 推荐方案: **SQLite** (1.0版本)

**理由**:
1. 单文件部署，无需安装数据库
2. 对于预期数据量(10万条记录)完全够用
3. 后续可平滑迁移到PostgreSQL

**迁移策略**: 使用GORM抽象数据库访问，1.x版本可无缝切换数据库

## 4. 攻击模拟引擎

### 传统BAS模块

| 方案 | 描述 | 评估 |
|------|------|------|
| 自研 | 从零开发攻击库 | 工作量大，但可控 |
| 集成Nuclei | 使用Nuclei模板 | 模板丰富，Go原生 |
| 集成OWASP ZAP | 调用ZAP API | 功能强大，但重 |

**推荐**: 1.0版本**自研+参考Nuclei模板格式**

### LLM安全模块

**直接复用 llm-security-bas**:
- 58个攻击用例
- 完整的执行引擎
- 关键词判定系统
